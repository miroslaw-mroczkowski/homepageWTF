Frontend i Backend

Czym jest Frontend

Frontend jest tym co widać w przeglądarce internetowej, youtube, Visual Studio Code.

Frontendowiec to:

  • Frontend Developer
  • JavaScript Developer
  • CSS/HTML Developer
  • Content Developer
  • Full-Stack Developer
  • UI Engineer
  • UI Developer
  • Webmaster

Puntkem wspólnym łączącym powyższe stanowiska są technologie używane do budowania aplikacji i stron:

  • HTML
  • CSS
  • JavaScript
Czym jest backend?

Backend jest wszystkim co się dzieje poza naszą przeglądarką (w dużym uproszczeniu. Patrząc na samochód, backend odpowiada za silnik i wszystkie układu, które sprawiają, że samochód jedzie. Frontend za karoserię i wnętrze używane przez kierowcę i pasażerów.

Podsumowanie

Frontend odpowiada za:

  • wygląd stron/aplikacji internetowych
  • odpowiada za interakcje z użytkownikiem
  • jest generowany głównie przy użyciu trzech technologii: HTML, CSS i JavaScript
  • technologie frontendowe przejmują rynek tradycyjnych aplikacji.

HTML - HyperText Markup Language

HTML jest językiem znaczników i pdowiada za treść, czyli za:

  • Teksty
  • Nagłówki
  • Linki
  • Listy
  • Definicje
  • Obrazki
  • Video
  • Formularze
  • Wszystko co jest nośnikiem informacji

Budowa znaczników

<znacznik atrybut="wartość">Treść</znacznik>

Semnatyczny HTML - html mający znaczenie. Rozszerzenie plików html to ".html"

Podstawowe znaczniki HTML

  • <main> - główna część strony
  • <section> - sekcja (np. sport w gazecie)
  • <article> -samodzielne treści
  • <header> - nagłówek dokumenty sekcji lub artykułu
  • <footer> - stopka dokumentu, sekcji bądź artykułu
  • <h1-h6> - nagłówki, od najważniejszego "h1", do najmniej ważnego "h6"
  • <p> - paragraf/akapit

Podusmowując, HTML:

  • Opakowuje i opisuje treść strony
  • Mówi przeglądarce o strukturze dokumentu
  • Posiada swoją zdefiniowaną składnię
  • Nie jest językiem programowania, jest językiem znaczników (Markup Language)
  • Aktualną rekomendowaną wersją jest 5.2
  • Cała specyfikacja jest na stronie stronie w3c

CSS - kaskadowe arkusze stylów (Cascading Style Sheets)

CSS odpowiada za prezentację oraz wpływa na wygląd elementów HTML na ekranie, wydrukach itp. Pliki zawierającee kod CSS mają rozszerzenie ".css". CSS ma swoją składnie:

Sposoby stylowania:

  1. Znacznik "style" - umieszczany w znaczniku head
  2. Zewnętrzny arkusz stylów - przy użyciu znacznika <link href="style.css" rel="stylesheet" type="text/css" >
  3. Styl przy znaczniku z wykorzystaniem atrybutu style nazywamy stylem inline <znacznik style="color:yellow;" > treść </znacznik>

Właściwości CSS

  1. color
    • blue, red, brown itp.
    • #cccccc - kodem szesnatkowym
    • rgb (rgba) - red(0-255) green(0-255) blue(0-255)

Podsumowując, CSS:

  • Odpowiada za prezentację treści strony
  • Nie mają jednej wersji, potocznie mówi się, że aktualna to wersja 3
  • Przeglądarki potrafią różnie interpretować CSS
  • Cała specyfikacja jest na stronie w3c

Czym jest programowanie?

Programowanie to zamiana ludzkiego języka na język zrozumiały przez komputer. Każdy język programowania ma swoją składnię, gramatykę i ortografię.

Na świecie mamy ponad 700 języków programowania. W tym kursie zajęliśmy się tylko jednym - JavaScript.

Podsumowując, JS:

  • Odpowiada za interakcje
  • Pozwala na programowanie we frontendzie
  • Potrafi wpływać na każdą z warstw
  • Rozszerza możliwości zwykłych stron
  • Aktualnie jest jednym z najpopularniejszych języków programowania na świecie

Tydzień III

Kanban

Kanban służy do planowania pracy, aby łatwo śledzić postępy. Z japońskiego oznacza spis widoczny

Wygląd kanbanu

ToDo = do zrobienia

Doing = w trakcie

Tylko jedno zadanie

Done = zrobione

Podsumowując, Kanban:

  • Ułatwia śledzenie postępu prac/nauki
  • Zaleca się trzymanie jednego zadania w "doing"
  • Dobrze opisuj "definition of done"
  • Można stosować offline
  • Online za darmo można przygotować swoją tablicę na Trello.com

HTML

Podstawowe znaczniki HTML

  • <br> - nowa linia
  • <b> oraz <i> - pogrubienie oraz kursywa, bez znaczenia semantycznego
  • <strong> oraz <em> - pogrubienie (podkreśla wysoką istotność frazy w nim zawartej) oraz kursywa (podkreśla, że tekst może mieć znaczenie (np. ironia)
  • <a href="" target="">opis łącza</a> - a- linki / łącza wewnątrz dokumentu
  • <element id="moj identyfikator"> - atrybut id - identyfikuje element (przydaje się do nawigacji w obrębie dokumentu

  • <nav> - nawigacja strony / menu
  • <img src="" alt=""> - obrazek o podanym adresie. src - adres do obrazka, alt - tekst alternatywny

Podusmowując:

  • atrubuty są "niewidzialne" - chowamy je w znaczniku
  • nadają elementom dodatkowe właściwości (href, src)
  • pozwalają na grupowanie elementów (class)
  • pozwalają na identyfikowanie elementów (id)
  • można je miksować
  • spis atrybutów "globalnych" znajdują się na stronie

CSS

Stylowanie za pomocą id i class

  • Stylowanie po class
    <znacznik class="name">treść</znacznik>

    .name {

    właściwość: wartość;

    }
    .(kropka) - selektor klasy
  • Stylowanie po id
    <znacznik id="name">treść</znacznik>

    #name {

    właściwość: wartość;

    }
    #(hash) - selektor id

Specyficzność w CSS

Nazewnictwo klas

  • używaj pojedynczych słów z j. angielskiego
  • nazywaj rzeczy zgodnie z ich funkcją na stronie
  • startuj z małej litery, np. header nie Header
  • nie zaczynaj od cyfry
  • jeśli musisz użyć dwóch słów użyj tzw. kebab case
  • staraj się, aby element miał jedną klasę (nie zwiększaj specyficzności bez potrzeby)

Podsumowanie

  • html class="name" = css .name
  • html id="name" = css #name
  • stylujemy po klasach (bez zagnieżdżeń
  • nazywamy klasy po angielsku, zgodnie z przeznaczeniem
  • tworzymy klasy nawet kiedy mamy jeden element danego typu
  • specyficzność może dać popalić :)
  • Nie używamy !important i stylów inline'owych
